GitHub Pages 踩坑记

很早之前就听说过可以使用 GitHub Pages 搭建个人博客，一直迟迟未动。

一方面是缺乏执行力，另一方面是因为不会。

去年开始在微信公众号更新一些文章，坚持了许久后，执行力方面得到了很大的提升，于是又想尝试搭建个人博客，把发表在微信公众号的文章转移到这里。

其实直到开始搭建博客时，这个技能我依旧不会，不过在这个互联网如此便捷的时代，还有什么技能是查不到的吗？

具体的教学步骤欢迎参考这篇博文，我也是一步步按照这篇文章的步骤开始搭建的。

GitHub Pages + Hexo搭建个人博客网站，史上最全教程

完全参考这篇文章也不会那么一帆风顺，毕竟每个人的电脑环境不一样，GitHub 也会逐渐的升级迭代，过去实用的规则现在可以已经废弃。

接下来再补充几点我遇到的问题。

  1. 安装 hexo 时无法将文件写入本地，需要在执行命令时添加 sudo，赋予本次命令高等级权限，可以执行写操作

  2. 将本地编译好的博客网页 push 到 GitHub 仓库时提示主机不识别。

    我们前往 /Users/用户/.ssh 目录下，查看 know_hosts 文件，该文件会记录 GitHub 的 ip 地址信息；同时我们使用 ping 命令查看 GitHub 的网址对应的 ip，发现该 ip 竟然不在 know_hosts 文件列表中。

    使用 ssh-keygen -R github.com 命令将 GitHub 新的信息添加到该文件中，就可以解决该问题

  3. 如果项目中配置的 GitHub 地址是 https 链接，在 push 代码时会遇到提示，GitHub 已经废弃该方式了，那怎么办呢？将 GitHub 仓库地址改为 ssh 链接，即可解决该问题。

  4. 如果你想将 GitHub Pages 的域名替换成其他的，可以去阿里云、或者腾讯云购买一个域名，再配置好 DNS 解析，就可以输入自己购买的域名地址访问个人博客网页啦。

    可以参考这篇文章为你的hexo博客配置个性域名

  5. 这个时候基本就完事了，可是还有一点点瑕疵，还是 http 协议，浏览器总是提示不安全，那可不可以升级成 https 协议呢？

    当然可以，可以参考这篇文章申请一个免费的 SSL 证书10 分钟从 HTTP 切换到 HTTPS，保证个人博客安全性！

至此本次踩坑历程就结束了，每个人搭建时一定还会遇到我不曾遇到的问题，别烦躁，你遇到的问题别人一定遇到过，耐心的从网上搜索一下解决方案，再稍稍适配一下我们的实际情况，问题不就解决了嘛。
